## Phase 1: SD Turbo → Z-Image-Turbo 切換

### 問題
- SD Turbo 生成圖片模糊，品質不佳
- 4 張圖 × 50s = 200s 超過 120s timeout

### 修復
- `sd_service/main.py` — 改用 `z_image_turbo_Q6_K.gguf`（5.3GB）+ `Qwen3-4B-Instruct` LLM（2.5GB）+ VAE（335MB）
- CFG scale: 7.0 → **1.0**（flow-matching model 必須用 CFG=1.0）
- numImages: 4 → 1，timeout: 2min → **10min**
- Steps: 4 → 8

---

## Phase 2: Q&A 系統全面重構

### 問題
- LLM 返回無效 field name（"detail"、"custom"）導致卡死
- 簡單物件（香蕉）和複雜物件（醫療櫃）問一樣多題
- 只問 ~2 題就結束

### 修復
- **LLM 驅動自適應提問** — LLM 分析已知欄位，決定下一個問題或 done
- **Hard-filter 驗證** — field 必須在 `[material, color, dimensions, shape, surface, edges, components, style, features]` 中，無效 → template fallback
- **Done 信任檢查** — 只信任 "done" 當 3+ 欄位已填
- **自適應難度** — 簡單物件 ~4 題，複雜物件 ~10 題
- **覆蓋率追蹤** — `field-tiers.ts`（優先級）+ `coverage.ts`（覆蓋率計算）
- **問題庫** — 依 assetType 提供不同問題模板
- **智能自由輸入解析** — 前端檢測顏色/材質/尺寸關鍵字，一次填入多欄位

---

## Phase 3: Prompt 潤色重構

### 問題
- Positive prompt 只有模板輸出，LLM 潤色悄悄失敗
- Negative prompt 完全沒潤色，永遠是固定的 `text, watermark, logo...`

### 修復
- **Positive polish** — 詳細 LLM prompt 含範例，放寬接受門檻（30+ chars），console log 成功/失敗
- **Negative polish** — 新增！LLM 根據物件材質/顏色/形狀/表面生成針對性負面詞
- Template 輸出作為 fallback

---

## Phase 4: 刪除 Hunyuan API

### 問題
- `.env` 設了 `HUNYUAN_BASE_URL=http://localhost:8080/v1` 但沒服務在跑
- 每次生圖都先嘗試連 Hunyuan → 掛 120 秒 timeout → 才 fallback 到 local SD
- 4 張圖 = 8 分鐘乾等

### 修復
- `src/lib/hunyuan/client.ts` — 刪除全部 Hunyuan 代碼（`callHunyuanT2I`、`callHunyuanI2T3D`、fallback chain）
- `textToImage()` 直連 local SD
- `src/lib/hunyuan/types.ts` — 移除 `HunyuanConfig`
- `.env` — 刪除 5 行 Hunyuan 設定

---

## Phase 5: 多視角生成（4 Views）

### 功能
- `multiView:true` → 生成 4 張圖：front/back/left/right orthographic views
- 每個視角獨立 prompt + negative：
  - Front: `front orthographic view, product photography, [base]` + exclude side/back
  - Back: `back orthographic view, rear view, product photography, [base]` + exclude front/drawers
  - Left/Right: side orthographic + exclude front/back/3/4 angle
- Base prompt 先清理方向性詞彙（`front-facing`、`centered`、`facing camera`）
- 順序生成（VRAM 限制）

## Phase 6: 四種 Input 模式全打通
