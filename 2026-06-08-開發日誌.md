## Phase 1: Q&A 流程重構
### 問題
- Question bank 的針對性問題從未真正顯示（bankQs 被建立但只在 validQs < 2 時才使用）
- 材質在第一輪就被問到，而不是最後 AI 推薦
- 選項中有多餘的「自訂」和「跳過」（已有輸入框和跳過按鈕）

### 修復
- Bank questions 在 filledCount >= 3 時完全取代 LLM 問題
- 從 VALID_FIELDS 移除 "material"，Q&A 不再問材質
- 所有選項移除「自訂」「Custom」「跳過」「Skip」「Other」「其他」
- LLM prompt 更新：不問材質、不生成多餘選項、尺寸必須 axbxc 格式
- 終止條件從 "5 fields" 改為 "all applicable fields"

## Phase 2: 材質推薦修復
### 問題
- LLM 永遠推薦 PLA（prompt 範例 JSON 寫了 `"material":"PLA"`）
- 沒有分析需求，直接複製範例

### 修復
- 範例改為 `"material":"PETG"`
- 新增強制分析步驟：「先分析需求：承重？碰水？耐熱？接觸食品？」
- 明確指令：「不要預設推薦PLA——根據實際需求判斷」

## Phase 3: Vision + Annotation 修復
### 問題
- Z-Image 生成 768px PNG（base64 ~1MB），Ollama vision API 拒絕
- Vision prompt 太公式化，LLM 回 "Hey there!" 而不是 JSON

### 修復
- 用 sharp 將圖片縮放至 512px JPEG（~21KB）
- 客戶端也加入 resizeImageForUpload（annotation 前先縮放）
- Vision prompt 改為自然對話風格但強制 JSON 輸出
- Annotation 改用 modify 模式（保留原有元素）

## Phase 4: 圖片生成品質
### 問題
- 解析度鎖在 768px，推理步數固定 8
- 批量生圖進度是模擬的，不準確

### 修復
- client.ts 解鎖 1024px
- UI 加入解析度 slider（512/768/1024）+ 步數 slider（8-30，預設 12）
- 改為逐一生成（每張完成立即顯示）+ 終止按鈕

## Phase 5: Iterate + Annotation 保留原有元素
### 問題
- 文字 iterate 和畫圖 annotation 都從頭生成新 prompt，丟失原有細節

### 修復
- prompt-craft.ts 新增 `buildModifyPrompt(existingPrompt, feedback)`
- craft/stream route 支援 modify 模式
- handleIterate 發送 existingPrompt + feedback
- annotation route 改用 buildModifyPrompt

## Phase 6: 短輸入過度清除修復
### 問題
- "I want a blue container" (24 字 < 30) → safety net 清除所有欄位（包括 "blue"）

### 修復
- 改用 `fieldMentioned()` 檢查提取值是否在用戶原文中出現
- 只清除 LLM 幻覺的欄位，保留用戶明確說的內容

## Phase 7: UI 暖銅質感重設計
### 配色
- 背景: #FDF8F3 暖奶油色 + SVG 紋理
- 主色: #C4823B 暖銅 / #A0522D 紅棕
- 文字: #2C2416 / #5C4A3A / #8B7355 / #B8A898

### 頁面
- 首頁：語言選擇屏 + 品牌 "Atelier" + 暖銅漸變
- Dashboard：簡化（移除統計/狀態/步驟），新增批量選取、排序、內聯編輯
- Create：全面色調更新（sed 批量替換 Tailwind class → 自訂色）
- 專案頁：重新設計（圖片網格 + 選取下載 + 重新打開專案）

## Phase 8: 中英文切換修復
### 問題
- UI placeholder 用 `cl`（convLang || toggleLang），extract 後 convLang 鎖定，切語言無效

### 修復
- UI 層文字改用 `toggleLang`，對話訊息保持 `cl`
