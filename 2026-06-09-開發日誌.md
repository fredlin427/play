# 2026-06-09 開發日誌 — 品質全面優化 + 自我進化系統

## 概述
今天聚焦在三個方向：(1) 程式碼品質優化 — 消除重複、拆分巨型組件、加入測試；(2) 生成品質提升 — craft 簡化、few-shot 升級、視覺模型更換；(3) 自我進化系統 — star 標記機制讓 AI 從使用者偏好中學習。

### Error Boundary

- `src/hooks/use-session.ts` — localStorage 持久化 hook

### prompt-helper.ts ask() 重構
- 300 行 → 130 行核心邏輯
- 提取 `buildKnownSummary()`, `buildBankQuestions()`, `mapBankField()`, `applyHardMinimum()`, `shouldTerminate()`

### 其他

## Phase 2: 測試
- 安裝 vitest，加入 `npm test` script
- 4 個測試檔、58 個測試用例，100% 通過
- 測試範圍：`extractJson`, `cleanPositive`, `validatePolish`, `applyPromptImprovements`, `getQuestionBank`, `getMaxRounds`, `getField`, `setField`, `isFieldFilled`, `isRedundantOpt`

## Phase 3: 生成品質提升
### Craft 簡化
- `buildJointPrompt`: 2000 字 → 400 字，砍掉 self-critique 6 步驟
- Craft route: 砍掉 validation + repair 循環（1-2 次多餘 LLM 呼叫）
- Craft 耗時: 5-8s → 1-3s

### Few-shot 升級
- 4 個內建範例全面重寫，加入精確尺寸、空間位置、材質描述
- 每個 prompt 結尾固定加 `single object isolated on pure white background, studio soft even lighting, product photography, 3D render quality`
- 負向 prompt 從 8-10 個擴到 10-15 個，分五大類（材質、顏色、幾何、部件、表面）

### 材質推薦升級
- Prompt 改用場景分類（醫療→PETG/Nylon、彈性→TPU、機械→ABS/Nylon）
- 智慧 fallback：LLM 掛掉時根據 spec 關鍵字推斷，不再一律 PLA
- 測試驗證：手術刀柄→Nylon、筆筒→PETG、手機殼→TPU

### 草圖分析修復
- Vision prompt 砍掉所有帶形狀名稱的範例（heart shape 等）— llava 會把範例當指令
- 簡化 prompt 從 1500 字 → 200 字，只問物件名+形狀
- 前端：用戶備註優先於 vision 輸出

### 標記分析修復
- Vision prompt 移除 "They drew a heart shape..." 範例
- 改為要求如實描述看到的形狀

### 視覺模型升級
- llava:7b (4.7GB) → qwen2.5vl:7b (6.0GB)
- 空間理解更強、繁體中文原生支援、支援 bounding box 輸出

## Phase 4: Q&A 分類強化
### 問題
- assetType 永遠是 "product"，因為 extract prompt 只有 2 行模糊提示
- Bank 問題在 filledCount >= 3 時完全取代 LLM，第 4 輪以後永遠同一套

### 修復
- Extract prompt 每個類型 30+ 關鍵詞（手術刀→medical, 齒輪→robot, 戒指→jewelry）
- Bank 不再取代 LLM，只在 LLM 回應不足時補充
- Asset hints 從一句變四句具體方向 + 排除條件
- 每輪問題從 2-3 → 3-4，最大輪次 medical 12 / product 10

## Phase 5: 自我進化系統（Star）
- Prisma schema: PromptVersion 加 `starred` 欄位
- `POST /api/prompt/star` — toggle 標記
- Craft 時讀取已標記 prompt，當作額外 few-shot example
- 前端：prompt 旁加 ⭐ 按鈕

## Phase 6: 上傳修復
- 圖片/3D 檔案模式：之前需要先有 projectId 才能上傳，但 projectId 要點 Start 才創建 → 死循環
- 修復：上傳時自動創建 project，不再需要先打字

## Phase 7: Docker 基礎建設
- `docker-compose.yml` — Ollama + SD Service + Next.js
- `Dockerfile` — Next.js 容器
- `sd_service/Dockerfile` — SD Service 容器
- 註：SD Service Docker GPU 支援有限，建議只用 Docker 跑 Ollama

## Phase 8: 版本歷史修復
- 點「還原」會新增一個「回調版本」→ 又出現還原按鈕 → 無限循環
- 修復：還原只切換內容，不再 recordVersion
