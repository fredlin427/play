- shadcn/ui 組件：Button、Card、Badge、Input、Textarea、Label、Separator、Skeleton、Tabs

**6. API 路由**

## 遇到的問題與解決

| 問題 | 解決方案 |
|------|---------|
| Prisma 7 `url` 不支援 | 改用 `prisma.config.ts` + JavaScript adapter |
| `datasourceUrl` TypeScript 報錯 | Prisma 7 必須用 `adapter` 選項 |
| PrismaClient 空建構子報錯 | 使用 `new PrismaBetterSqlite3({ url: "file:./dev.db" })` |
| tldraw v5 API 不相容 | 改用 `Tldraw` 組件 + `onMount` prop |
| `Cube` 不存在於 lucide-react | 改用 `Box` |
| PowerShell 測試中文變亂碼 | 必須用 `[Text.Encoding]::UTF8.GetBytes()` + `charset=utf-8` |
| Ollama 不遵循 system prompt 語言指令 | 改為整個 prompt 用目標語言（非附加指令） |
| 語言偵測不穩定（英文對話 + 中文訊息） | 門檻從 15% 降到 8%，權重 CJK×3 |
| PowerShell `$pid` 是唯讀變數 | 改用 `$projId` |
| Next.js 16 `params` 必須 await | 所有 API route 加上 `await params` |

---

## 學到的東西

1. **Prisma 7 有重大 breaking changes**：需要 driver adapter，url 移到 config
2. **Next.js 16**：params 是 Promise，路由型別生成有變化
3. **Tailwind v4**：無 config 檔案，用 CSS `@theme inline` 設定
4. **Ollama 部署**：小模型（3B）適合開發，大模型（14B）適合生產
5. **tldraw v5 API**：與 v2 完全不同，需要查最新文檔
6. **語言偵測**：權重系統比簡單比例更好，特別是中英夾雜場景
7. **Zod + retry + fallback** 三層防禦對於 LLM 輸出非常有效
8. **FastAPI + Next.js** 微服務架構適合需要 Python 生態的場景
