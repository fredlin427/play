# 2026-06-04 開發日誌 — Hunyuan 2D→3D Pipeline 重構

## 概述

將原 Hospital 3D Print AI 從醫院專用 parametric CAD pipeline 完全重構為通用 **文字→2D圖片→3D模型→Blender後處理→STL** 工作流。

|------|---------|
| OS | Windows 11 Pro 10.0.26100 |
| Node.js | v24.16.0 |
| Python | 3.13.4 |
| CUDA | 13.2 (RTX 4060 Laptop 8GB) |
| Ollama 模型 | qwen2.5:3b (預設) / qwen3.5:4b / qwen2.5:14b |

## Phase 1: 資料庫重構

- **刪除**舊 hospital-specific 欄位：riskLevel / structuredRequirements / designBrief / materialRecommendation / ticket / sketchData / designState / cadTemplate / stlFilePath
- **新增** 6 個 Models：`PromptVersion` / `ReferenceImage` / `ReferenceModel` / `GeneratedImage` / `GeneratedModel` / `BlenderJob`
- Project 狀態改為工作流狀態鏈：`draft → prompt_crafting → image_review → model_review → blender_processing → engineer_review → stl_ready → completed`
- Prisma 7 + SQLite，備份 dev.db 後 re-push

## Phase 2: 核心邏輯

- `src/lib/hunyuan/client.ts` — Hunyuan API 客戶端，含三層 fallback：Hunyuan → 本地 SD → Mock
- `src/lib/blender/client.ts` — Blender subprocess 管理，含 mock 模式
- `src/lib/agents/prompt-helper.ts` — 簡化為兩個獨立的 LLM 呼叫：`analyze()` + `craft()`
- `src/lib/storage.ts` — 檔案管理（images/ / thumbnails/ / models/ / stl/）
- `blender_scripts/auto_process.py` — Blender 無頭自動處理（mesh修復 + 可列印性檢查 + STL匯出）
- `sd_service/` — 本地 Stable Diffusion Turbo (FastAPI, port 8001, CUDA 加速)

| POST `/api/upload/image` | 上傳參考圖片 |

## Phase 6: 關鍵問題修復

### 問題 1: qwen3.5:4b thinking model 導致 LLM 失敗
- **症狀**：LLM 呼叫 63 秒後回傳 fallback
- **原因**：qwen3.5 將回應放在 `reasoning` 欄位而非 `content`，且 reasoning 內的 JSON 被 greedy regex 錯誤匹配
- **修復**：
  - `src/lib/llm.ts` — 讀取 `reasoning` 當 `content` 為空時
  - `src/lib/llm.ts` — 重寫 `extractJson()`：找出所有 `{...}` blocks，從最後一個開始解析，找到有效 JSON 即返回
  - 最終換回 `qwen2.5:3b` 作為預設（更快 1.6s vs 63s，JSON 更可靠）

### 問題 2: 中英文切換混亂
- **症狀**：中文輸入 → AI 回英文；回答 "50-100mm" → 切換成英文
- **原因**：`detectLang()` 只看最後一條訊息（短英文無 CJK）
- **修復**：改用全部對話歷史檢測語言，一旦中文鎖定中文

### 問題 3: 圖片顯示不出來
- **症狀**：生成的 2D 圖片在 `/projects/[id]` 頁面無法顯示
- **原因**：Windows 路徑 `\` vs url `/` 不匹配，`split("uploads/")` 在 Windows 無效
- **修復**：新增 `extractUploadPath()` 函數，同時處理 `/` 和 `\`

### 問題 4: 提示詞輸出千篇一律
- **症狀**：不同用戶輸入 → 相同模板提示詞
- **修復**：系統提示詞強化「必須使用用戶確切詞彙」「禁止通用模板」，temperature: 0.5

### 問題 5: 問答流程無限循環
- **症狀**：AI 重複問相同問題，問了 10+ 輪不結束
- **修復**：改為前端驅動模式 — 5 個預定義問題模板，前端追蹤已回答/未回答，全部完成後自動顯示「生成提示詞」按鈕。LLM 只做簡單的 analyze + craft
