# 2026-06-04 開發日誌 — Hunyuan 2D→3D Pipeline 重構

## 概述

將原 Hospital 3D Print AI 從醫院專用 parametric CAD pipeline 完全重構為通用 **文字→2D圖片→3D模型→Blender後處理→STL** 工作流。

## Phase 1: 資料庫重構

## Phase 6: 關鍵問題修復

### 問題 1: qwen3.5:4b thinking model 導致 LLM 失敗
- **症狀**：LLM 呼叫 63 秒後回傳 fallback
- **原因**：qwen3.5 將回應放在 `reasoning` 欄位而非 `content`，且 reasoning 內的 JSON 被 greedy regex 錯誤匹配
- **修復**：
  - `src/lib/llm.ts` — 讀取 `reasoning` 當 `content` 為空時
  - `src/lib/llm.ts` — 重寫 `extractJson()`：找出所有 `{...}` blocks，從最後一個開始解析，找到有效 JSON 即返回
  - 最終換回 `qwen2.5:3b` 作為預設（更快 1.6s vs 63s，JSON 更可靠）

### 問題 2: 中英文切換混亂
- **症狀**：中文輸入 → AI 回英文；回答 "50-100mm" → 切換成英文
- **原因**：`detectLang()` 只看最後一條訊息（短英文無 CJK）
- **修復**：改用全部對話歷史檢測語言，一旦中文鎖定中文

### 問題 3: 圖片顯示不出來
- **症狀**：生成的 2D 圖片在 `/projects/[id]` 頁面無法顯示
- **原因**：Windows 路徑 `\` vs url `/` 不匹配，`split("uploads/")` 在 Windows 無效
- **修復**：新增 `extractUploadPath()` 函數，同時處理 `/` 和 `\`

### 問題 4: 提示詞輸出千篇一律
- **症狀**：不同用戶輸入 → 相同模板提示詞
- **修復**：系統提示詞強化「必須使用用戶確切詞彙」「禁止通用模板」，temperature: 0.5

### 問題 5: 問答流程無限循環
- **症狀**：AI 重複問相同問題，問了 10+ 輪不結束
- **修復**：改為前端驅動模式 — 5 個預定義問題模板，前端追蹤已回答/未回答，全部完成後自動顯示「生成提示詞」按鈕。LLM 只做簡單的 analyze + craft

## Phase 7: Prompt 生成管線全面修復 (17:00-18:00)

### 問題診斷
發現 prompt 生成（尤其是 craft 環節）輸出品質極差，經分析確定 **9 個問題**。

### 修復 1: CRAFT 系統提示詞重寫 (`prompts.ts`)
- **前**：`Generate 9-section prompt from spec.`（一句話）
- **後**：50 行詳細規範，明確定義 9 個 section 各是什麼、格式要求、6 條規則
- 9-section 結構：Object Name → Positive Prompt → Negative Prompt → Key Visual Features → Material → Geometric Structure → View → Dimensions → Generation Notes

### 修復 2: Regex 後處理改為標題文字匹配 (`prompt-helper.ts`)
- **前**：硬編碼假設 section 2 = positive, section 3 = negative
- **後**：按標題文字 `Positive Prompt` / `Negative Prompt` 匹配，附 legacy fallback

### 修復 3: Feedback 傳遞修復 (`craft/route.ts` + `prompt-helper.ts`)
- **前**：前端傳了 `feedback` 參數但後端完全忽略
- **後**：`craft()` 新增 `feedback?` 參數，拼接到 user message

### 修復 4: Mock 模式輸出格式 (`llm.ts`)
- `mockPrompt()` 重寫為完整 9-section markdown 格式，與真實 LLM 輸出一致
- mock dispatcher 新增匹配關鍵詞 `prompt engineer` / `9-section prompt`

### 修復 5: ask() 改用 callLLMStructured (`prompt-helper.ts` + `schemas.ts`)
- **前**：`callLLM()` + 手動 regex JSON 解析，無 retry
- **後**：`callLLMStructured` + `AskQuestionArraySchema`，自動 retry/repair
- 新增 `ASK_ARRAY_FALLBACK`（取代舊的單一 `ASK_FALLBACK`）

### 修復 6: Extract 布爾值不再硬編碼 (`schemas.ts` + `prompts.ts` + `prompt-helper.ts`)
- ExtractSpecSchema 新增 `hasHoles` / `hasGrooves` / `hasMovingParts` / `isHollow` / `poseOrOrientation`
- Extract prompt 教 LLM 推斷這些值（如 "hollow tube" → isHollow:true）

### 修復 7: styleNotes 截斷 (`craft/route.ts`)
- **前**：`result.content.slice(0, 200)` — 只存 200 字符
- **後**：擴大為 2000 字符

### 修復 8: Thinking model 支援 (`prompt-helper.ts`)
- qwen3.5 等思考型模型輸出的 CoT 文字會被剝離後才進行 section 解析
- `maxTokens` 從 2500 提高到 4096（思考型需要更多）
